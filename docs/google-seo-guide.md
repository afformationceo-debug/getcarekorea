# Google SEO 기본 가이드 (RAG Reference)

> 출처: https://developers.google.com/search/docs/fundamentals/seo-starter-guide?hl=ko
> 최종 업데이트: 2026-01-22
> 용도: LLM RAG 참조용 - 콘텐츠 생성 시 항상 참고

## 1. Google 검색 작동 원리

Google은 완전히 자동화된 검색엔진으로, 크롤러(웹 크롤러)라는 프로그램을 사용하여 끊임없이 웹을 탐색하며 색인에 추가할 페이지를 찾습니다.

**핵심 프로세스:**
1. **크롤링**: 웹 크롤러가 웹을 탐색하며 페이지 발견
2. **색인 생성**: 페이지 콘텐츠 분석 및 색인 저장
3. **검색 결과 제공**: 사용자 쿼리에 가장 관련성 높은 결과 표시

## 2. Google이 콘텐츠를 발견하도록 하기

### 색인 여부 확인
```
site:example.com
```
이 검색 연산자로 사이트가 Google 색인에 포함되어 있는지 확인합니다.

### 발견 방법
- **자동 발견**: 다른 페이지의 링크를 따라가며 자동 발견 (가장 일반적)
- **사이트맵 제출**: `sitemap.xml` 파일로 페이지 목록 제공 (선택사항)
- **Search Console**: Google Search Console에서 URL 제출

### 권장사항
- 새 페이지는 기존 페이지에서 링크 연결
- 사이트맵은 동적으로 생성하여 최신 상태 유지

## 3. 사이트 구성 전략

### URL 설계 원칙
✅ **좋은 예:**
```
https://example.com/procedures/facial-contouring
https://example.com/hospitals/gangnam/clinic-name
```

❌ **나쁜 예:**
```
https://example.com/index.php?id_sezione=360&sid=3a5ebc944f41daa6f849f730f1
```

**URL 베스트 프랙티스:**
- 설명적인 단어 사용 (숫자/ID 대신)
- 하이픈(-)으로 단어 구분 (밑줄 피하기)
- 단순하고 읽기 쉽게 유지
- 키워드 포함 (자연스럽게)

### 디렉터리 구조
```
/procedures/          # 시술 카테고리
  /facial/           # 얼굴 시술
  /body/             # 바디 시술
/hospitals/          # 병원
  /seoul/            # 지역별
  /busan/
/blog/              # 블로그
  /guides/          # 가이드
  /news/            # 뉴스
```

**원칙:**
- 주제별로 유사 페이지 그룹화
- 논리적인 계층 구조
- 깊이는 3-4단계 이하 권장

### 중복 콘텐츠 처리

**표준 URL 지정:**
```html
<link rel="canonical" href="https://example.com/procedures/facial-contouring" />
```

**리디렉션:**
- 301 리디렉션으로 중복 URL을 표준 URL로 통합
- 예: www 있음/없음, http/https 통일

## 4. 콘텐츠 품질 원칙

### E-E-A-T 원칙
Google은 **E-E-A-T** 기준으로 콘텐츠 품질을 평가합니다:
- **Experience (경험)**: 실제 경험을 바탕으로 작성
- **Expertise (전문성)**: 주제에 대한 전문 지식
- **Authoritativeness (권위성)**: 해당 분야에서 신뢰할 수 있는 출처
- **Trustworthiness (신뢰성)**: 정확하고 투명한 정보

### YMYL (Your Money Your Life) 콘텐츠
의료, 건강, 금융 등 사용자의 삶에 영향을 주는 주제는 더 높은 품질 기준 적용:
- 자격을 갖춘 전문가가 작성
- 정확한 정보, 출처 명시
- 최신 상태 유지
- 면책조항 제공

### 콘텐츠 작성 원칙

**필수 요소:**
1. **읽기 쉬운 텍스트**
   - 명확하고 간결한 문장
   - 논리적인 단락 구성
   - 제목과 소제목으로 구조화
   - 글머리 기호와 번호 목록 활용

2. **고유한 콘텐츠**
   - 다른 사이트 복사 금지
   - 자체 사이트 내에서도 중복 최소화
   - 독창적인 관점과 분석 제공

3. **최신 상태 유지**
   - 오래된 정보 업데이트
   - 발행일/수정일 표시
   - 계절성 콘텐츠 주기적 갱신

4. **사용자 중심 작성**
   - 검색엔진이 아닌 사람을 위해 작성
   - 사용자 의도 충족
   - 질문에 명확한 답변 제공

**피해야 할 사항:**
❌ 키워드 반복 (키워드 스터핑)
❌ 자동 생성된 품질 낮은 콘텐츠
❌ 숨겨진 텍스트나 링크
❌ 게이트웨이 페이지 (도어웨이)
❌ 스크래핑된 콘텐츠

## 5. 검색 결과 표시 최적화

### 제목 링크 (Title Link)

제목 링크는 `<title>` 요소에서 주로 생성됩니다.

**베스트 프랙티스:**
```html
<title>서울 강남 안면윤곽 성형외과 | 병원명 - GetCare Korea</title>
```

**원칙:**
- **명확하고 간결**: 50-60자 권장
- **고유성**: 각 페이지마다 다른 제목
- **정확한 설명**: 페이지 내용을 정확히 반영
- **브랜드 포함**: 끝에 사이트명 추가 (선택)
- **키워드 앞쪽 배치**: 중요한 키워드를 제목 앞부분에

❌ **피해야 할 사항:**
- "페이지", "홈페이지" 같은 일반적인 텍스트
- "키워드, 키워드, 키워드" 같은 나열
- 모든 페이지에 동일한 제목

### 스니펫 (Snippet)

스니펫은 검색 결과에 표시되는 페이지 설명입니다.

**메타 설명 작성:**
```html
<meta name="description" content="서울 강남 최고의 안면윤곽 전문 성형외과. 3D CT 분석, 20년 경력 전문의, 한국어-영어 통역 지원. 무료 상담 예약 가능.">
```

**원칙:**
- **짧고 명확**: 150-160자 권장
- **고유성**: 각 페이지마다 다른 설명
- **관련성**: 페이지의 핵심 내용 요약
- **행동 유도**: "무료 상담", "지금 예약" 등
- **키워드 포함**: 자연스럽게 주요 키워드 포함

**주의사항:**
- Google은 페이지 콘텐츠에서 더 나은 설명을 자동 생성할 수 있음
- 메타 설명은 순위에 직접 영향 없음 (CTR에는 영향)

### 구조화된 데이터

Google에 콘텐츠를 더 명확히 이해시키기 위해 구조화된 데이터(Schema.org) 사용:

**의료/병원 관련:**
```json
{
  "@context": "https://schema.org",
  "@type": "MedicalBusiness",
  "name": "병원명",
  "description": "설명",
  "address": {...},
  "medicalSpecialty": "PlasticSurgery"
}
```

**블로그 포스트:**
```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "제목",
  "author": {
    "@type": "Person",
    "name": "작성자명"
  },
  "datePublished": "2026-01-22",
  "dateModified": "2026-01-22",
  "image": "이미지 URL"
}
```

**FAQ:**
```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "@type": "Question",
    "name": "질문",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "답변"
    }
  }]
}
```

**HowTo (가이드):**
```json
{
  "@context": "https://schema.org",
  "@type": "HowTo",
  "name": "제목",
  "step": [{
    "@type": "HowToStep",
    "text": "단계 설명"
  }]
}
```

## 6. 이미지 최적화

### 베스트 프랙티스

1. **고화질 이미지 사용**
   - 선명하고 전문적인 이미지
   - 적절한 해상도 (너무 크거나 작지 않게)

2. **관련 위치에 배치**
   - 이미지가 설명하는 텍스트 근처에 배치
   - 페이지 콘텐츠와 연관성 유지

3. **Alt 텍스트 작성**
```html
<img src="facial-contouring-before-after.jpg"
     alt="안면윤곽 수술 전후 비교 - 자연스러운 V라인 결과">
```

**Alt 텍스트 원칙:**
- 이미지 내용을 명확히 설명
- 짧고 간결 (125자 이하)
- 자연스럽게 키워드 포함
- "이미지", "그림" 같은 단어 불필요

4. **파일명 최적화**
```
✅ facial-contouring-surgery-korea.jpg
❌ IMG_1234.jpg
```

5. **이미지 포맷**
   - WebP: 최신 포맷, 압축률 우수
   - JPEG: 사진에 적합
   - PNG: 투명 배경, 로고/아이콘

6. **Lazy Loading**
```html
<img src="image.jpg" loading="lazy" alt="설명">
```

## 7. 동영상 최적화

### 베스트 프랙티스

1. **독립 페이지에 삽입**
   - 동영상마다 고유한 URL 페이지
   - 동영상이 메인 콘텐츠인 페이지

2. **제목과 설명 작성**
   - 설명적인 제목
   - 상세한 설명 (키워드 포함)
   - 타임스탬프 제공

3. **썸네일 최적화**
   - 고화질 썸네일 이미지
   - 내용을 잘 표현하는 장면

4. **트랜스크립트 제공**
   - 동영상 내용 텍스트 버전
   - 검색엔진이 내용 이해 가능
   - 접근성 향상

## 8. AEO (Answer Engine Optimization)

### AEO란?
Answer Engine Optimization은 AI 검색 엔진(Perplexity, ChatGPT Search 등)과 Google의 AI 요약 기능에 최적화하는 전략입니다.

### AEO 최적화 방법

1. **직접적인 답변 제공**
   - 질문에 명확한 답변을 첫 단락에 제공
   - "TL;DR", "요약" 섹션 추가

2. **구조화된 콘텐츠**
   - FAQ 형식 활용
   - 단계별 가이드 (HowTo)
   - 표와 목록으로 정보 구조화

3. **권위 있는 출처 인용**
   - 연구 논문, 공식 자료 링크
   - 전문가 인용
   - 통계 데이터 제공

4. **자연어 검색 최적화**
   - 질문 형태의 제목 사용
   - "무엇", "어떻게", "왜" 같은 키워드
   - 대화체 문장

예시:
```markdown
# 안면윤곽 수술 회복기간은 얼마나 걸리나요?

## 요약
안면윤곽 수술의 평균 회복기간은 2-3주입니다. 붓기가 가라앉는데 2주, 일상 복귀까지 3주, 완전한 회복은 3-6개월 소요됩니다.

## 단계별 회복 과정
1. **수술 직후 (1-3일)**: 심한 부기, 입원 관찰...
2. **1주차**: 부기 50% 감소, 실밥 제거...
3. **2-3주차**: 일상 복귀 가능...
```

## 9. 웹사이트 홍보

### 효과적인 홍보 방법

1. **소셜 미디어**
   - Facebook, Instagram, YouTube, TikTok
   - 시각적 콘텐츠 (전후 사진, 동영상)
   - 해시태그 전략

2. **커뮤니티 참여**
   - 의료 관광 포럼
   - Q&A 사이트 (Quora, Reddit)
   - 전문가로서 가치 제공

3. **입소문 마케팅**
   - 탁월한 서비스 제공
   - 고객 후기 적극 수집
   - 리뷰 프로그램 운영

4. **오프라인 홍보**
   - 명함, 브로셔에 웹사이트 주소
   - 이메일 서명에 링크
   - 뉴스레터

5. **협력 및 파트너십**
   - 관련 업체와 협력
   - 게스트 포스팅
   - 인플루언서 마케팅

## 10. 링크 전략

### 내부 링크

**베스트 프랙티스:**
```html
<a href="/procedures/facial-contouring">안면윤곽 수술 상세 정보</a>
```

**원칙:**
- 설명적인 앵커 텍스트 ("여기를 클릭" 피하기)
- 관련 페이지 간 연결
- 논리적인 사이트 구조 형성
- 고립된 페이지 없도록 관리

### 외부 링크

**신뢰할 수 있는 링크:**
```html
<a href="https://pubmed.ncbi.nlm.nih.gov/12345" target="_blank">
  연구 출처
</a>
```

**신뢰할 수 없는 링크:**
```html
<a href="https://unknown-site.com" rel="nofollow">
  링크 텍스트
</a>
```

**rel 속성:**
- `rel="nofollow"`: 링크를 따라가지 말라고 지시
- `rel="sponsored"`: 광고/스폰서 링크
- `rel="ugc"`: 사용자 생성 콘텐츠

### 사용자 생성 콘텐츠
댓글, 포럼 게시물 등의 링크에는 자동으로 `nofollow` 추가하여 스팸 방지

## 11. 무시해야 할 오래된 SEO 관행

| 오래된 관행 | 현재 상태 |
|-----------|---------|
| **메타 키워드 태그** | Google 검색에서 사용하지 않음 |
| **키워드 밀도 계산** | 스팸으로 간주, 자연스러운 사용 권장 |
| **도메인에 키워드 포함** | 순위에 미미한 영향 |
| **정확한 단어 수** | 길이 자체는 순위 요소 아님, 품질이 중요 |
| **PageRank만 집중** | 수백 가지 요소 중 하나일 뿐 |
| **과도한 링크 구축** | 자연스러운 링크 획득이 중요 |

## 12. 모바일 최적화

### 핵심 원칙

1. **반응형 디자인**
   - 모든 화면 크기에 적응
   - 단일 URL 사용

2. **빠른 로딩 속도**
   - 이미지 최적화
   - 불필요한 스크립트 제거
   - CDN 사용

3. **터치 친화적 인터페이스**
   - 충분한 버튼 크기 (48px 이상)
   - 적절한 여백
   - 읽기 쉬운 폰트 크기 (16px 이상)

4. **간소화된 내비게이션**
   - 명확한 메뉴 구조
   - 검색 기능 쉽게 접근
   - 햄버거 메뉴 활용

## 13. 페이지 속도 최적화

### Core Web Vitals

Google은 다음 지표로 사용자 경험을 평가합니다:

1. **LCP (Largest Contentful Paint)**
   - 주요 콘텐츠 로딩 시간
   - 목표: 2.5초 이하

2. **FID (First Input Delay)**
   - 첫 상호작용 반응 시간
   - 목표: 100ms 이하

3. **CLS (Cumulative Layout Shift)**
   - 레이아웃 안정성
   - 목표: 0.1 이하

### 최적화 방법
- 이미지 압축 및 lazy loading
- CSS/JS 최소화
- 서버 응답 시간 개선
- 브라우저 캐싱 활용
- CDN 사용

## 14. 국제화 및 다국어 SEO

### hreflang 태그

여러 언어 버전이 있을 때 사용:

```html
<link rel="alternate" hreflang="ko" href="https://example.com/ko/page" />
<link rel="alternate" hreflang="en" href="https://example.com/en/page" />
<link rel="alternate" hreflang="ja" href="https://example.com/ja/page" />
<link rel="alternate" hreflang="zh-CN" href="https://example.com/zh-cn/page" />
<link rel="alternate" hreflang="x-default" href="https://example.com/" />
```

**원칙:**
- 각 언어 버전에 고유한 URL
- 번역기 자동 번역 피하기
- 현지화된 콘텐츠 (단순 번역이 아닌)
- x-default로 기본 페이지 지정

### URL 구조

선택지:
1. **서브디렉터리**: `example.com/ko/`, `example.com/en/`
2. **서브도메인**: `ko.example.com`, `en.example.com`
3. **ccTLD**: `example.kr`, `example.com`

추천: 서브디렉터리 (관리 간편, SEO 권한 공유)

## 16. 콘텐츠 업데이트 전략

### 주기적 갱신

1. **에버그린 콘텐츠**
   - 연 1-2회 검토 및 업데이트
   - 통계, 데이터 최신화
   - 새로운 정보 추가

2. **뉴스/트렌드 콘텐츠**
   - 시의성 있을 때 발행
   - 빠른 색인 요청
   - 소셜 미디어 적극 활용

3. **업데이트 표시**
```html
<time datetime="2026-01-22">최종 업데이트: 2026년 1월 22일</time>
```

## 17. 검색 인텐트 (Search Intent) 이해

### 4가지 검색 의도

1. **정보형 (Informational)**
   - "안면윤곽 수술이란?"
   - 답변: 상세한 가이드, 설명 콘텐츠

2. **탐색형 (Navigational)**
   - "삼성서울병원 성형외과"
   - 답변: 특정 페이지 제공

3. **거래형 (Transactional)**
   - "안면윤곽 수술 예약"
   - 답변: 예약 페이지, CTA

4. **상업 조사형 (Commercial Investigation)**
   - "서울 최고 성형외과 비교"
   - 답변: 비교 리스트, 리뷰

### 인텐트에 맞는 콘텐츠

각 키워드의 검색 의도를 파악하고 적절한 콘텐츠 타입 제공
